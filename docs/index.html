RCE Dramatically Improves Hallucination Detection

When GPT-4 makes errors, RCE catches them with high accuracy

What This Benchmark Tests

This validation measures RCE's ability to detect when GPT-4 hallucinates (gives wrong answers). We test GPT-4 on difficult questions, then check if RCE can identify the errors.

  • LLM Accuracy: How often GPT-4 answers correctly without RCE
  • RCE Detection Rate: Of GPT-4's errors, how many does RCE catch?
  • Goal: RCE should detect >60% of hallucinations

Key Findings

  • F12 (Logical Fallacies): RCE detects 80% of GPT-4's errors - exceeding the 60% target
  • F11 (Complex Reasoning): RCE detects 67% of errors, even though it wasn't designed for this task
  • Model Tested: GPT-4 120B via Groq API (openai/gpt-oss-120b)
  • RCE Version: v0.6.0 with improved coherence validation
  • Validation: Real API calls, reproducible results

Reproduction Instructions

To reproduce these results with fixed prompts (no empty responses):

  1. Clone repository: git clone https://github.com/IsmaIkami/rce-gpt4-validation.git
  2. Install dependencies: pip install -r requirements.txt
  3. Start RCE API: cd rce-deployment-v0.6.0-api && python main.py
  4. Set Groq API key: export GROQ_API_KEY="your_key"
  5. Run benchmarks: python3 scripts/run_benchmarks.py

Note: Prompts now include "MUST always provide an answer" to prevent empty responses
